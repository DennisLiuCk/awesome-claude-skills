# Lead Research Assistant - 正體中文說明文件

## Skill 概述

### 功能
Lead Research Assistant skill 為您的產品或服務識別高品質潛在客戶。透過分析您的業務、搜尋目標公司並提供可行動的聯絡策略，完美適合銷售、業務開發和行銷專業人士。

### 解決的問題
業務開發人員在尋找潛在客戶時常面臨以下挑戰：

1. **不知從何開始**：不確定誰是理想客戶
2. **手動研究耗時**：逐一搜尋和評估公司非常費時
3. **缺乏脈絡資訊**：找到公司名稱但不知如何接觸
4. **優先順序不明**：所有潛在客戶看起來一樣重要
5. **個人化困難**：缺乏針對每個潛在客戶的具體資訊
6. **聯絡策略不清**：不知道該對誰說什麼

這個 skill 自動化潛在客戶研究過程，從理解您的產品到提供個人化的接觸策略。

## 核心功能分析

### 1. 理解您的業務

**分析面向**：
- 產品/服務的核心功能
- 價值主張
- 目標市場和受眾
- 解決的問題
- 競爭優勢

**資訊來源**：
- 用戶描述
- 程式碼庫分析（如果在專案目錄中運行）
- 網站或文件
- 產品定位說明

### 2. 識別目標公司

**搜尋標準**：
- **產業和領域**：特定市場區隔
- **公司規模和地點**：員工數、營收範圍、地理位置
- **技術棧和工具**：他們使用的技術
- **成長階段和募資**：新創、成長期、成熟企業
- **痛點**：您的產品解決的問題

**搜尋訊號**：
- 職缺發布
- 技術棧指標
- 最近新聞或公告
- 募資活動
- 擴張計畫

### 3. 優先級排序

**適配度評分（1-10）**：
基於以下因素評分：
- 與理想客戶檔案（ICP）的契合度
- 立即需求的訊號
- 預算可用性
- 競爭環境
- 時機指標

**分類**：
- 高優先級（8-10）：立即接觸
- 中優先級（5-7）：觀察和培養
- 低優先級（1-4）：長期跟進

### 4. 提供聯絡策略

**每個潛在客戶包含**：
- **公司名稱**和網站
- **為何適合**：基於他們業務的具體原因
- **優先級評分**：1-10 附說明
- **決策者**：要接觸的角色/職稱
- **聯絡策略**：個人化接觸建議
- **價值主張**：您的產品如何解決他們的特定問題
- **話題起點**：在接觸中提及的具體要點
- **LinkedIn URL**：如可用，便於連接

### 5. 資料豐富化

**收集資訊**：
- 決策者的姓名和職稱
- 公司背景和脈絡
- 最近新聞或動態
- 共同連結或推薦人
- 相關內容或興趣

### 6. 可行動輸出

**結構化報告格式**：
```markdown
# 潛在客戶研究結果

## 摘要
- 找到的潛在客戶總數：[X]
- 高優先級（8-10）：[X]
- 中優先級（5-7）：[X]
- 平均適配度評分：[X]

---

## 潛在客戶 1：[公司名稱]

**網站**：[URL]
**優先級評分**：[X/10]
**產業**：[產業]
**規模**：[員工數/營收範圍]

**為何適合**：
[基於他們業務的 2-3 個具體原因]

**目標決策者**：[角色/職稱]
**LinkedIn**：[URL 如可用]

**對他們的價值主張**：
[對該公司的具體好處]

**接觸策略**：
[個人化方法 - 提及具體痛點、最近公司新聞或相關脈絡]

**話題起點**：
- [具體要點 1]
- [具體要點 2]

---

[為每個潛在客戶重複]
```

## Claude Code 使用方式

### 觸發時機

Claude Code 會在以下情況使用此 Skill：

1. **基本使用**
   ```
   我正在建立[產品描述]。在[地點/產業]為我找 10 家適合的潛在客戶公司
   ```

2. **從程式碼庫運行**
   ```
   查看我在這個倉庫中建立的內容，識別[地點/產業]中前 10 家會受益於此產品的公司
   ```

3. **進階使用**
   ```
   我的產品：[描述]
   理想客戶檔案：
   - 產業：[產業]
   - 公司規模：[規模範圍]
   - 地點：[地點]
   - 當前痛點：[痛點]
   - 使用的技術：[技術棧]

   為我找 20 個合格的潛在客戶，每個都附聯絡策略
   ```

### 典型使用場景

#### 場景 1：來自 Lenny's Newsletter 的範例

**使用者**：「我正在建立一個在 AI 編程助手查詢中遮罩敏感資料的工具。找出潛在客戶。」

**輸出**：創建優先排序的公司清單，這些公司：
- 使用 AI 編程助手（Copilot、Cursor 等）
- 處理敏感資料（金融科技、醫療保健、法律）
- 在 GitHub 倉庫中有使用編程代理的證據
- 可能在程式碼中意外暴露敏感資料
- 包含相關決策者的 LinkedIn URL

#### 場景 2：地方企業

**使用者**：「我經營遠端團隊生產力諮詢業務。在灣區為我找 10 家最近轉為遠端的公司。」

**輸出**：識別以下公司：
- 最近發布遠端職缺
- 宣布遠端優先政策
- 正在招聘分散式團隊
- 顯示遠端工作挑戰跡象
- 為每家提供個人化接觸策略

#### 場景 3：SaaS 產品發布

```
為我的專案管理工具找潛在客戶。目標是 50-200 人的新創公司，正在快速成長並在招聘
```

**Claude 會**：
1. 搜尋符合規模標準的新創公司
2. 識別成長指標（招聘、募資）
3. 評估專案管理需求
4. 排序最有希望的潛在客戶
5. 為每家提供具體的接觸策略

## 使用的 Prompt 策略

### 1. 清晰的使用場景

明確說明何時使用：
- 為產品/服務尋找潛在客戶

1. 理解您的業務
2. 定義理想客戶檔案
3. 研究和識別潛在客戶
4. 優先級排序和評分
5. 提供可行動輸出
6. 提供後續步驟建議

展示完整的潛在客戶檔案結構。

## 最佳實踐

### 使用技巧

1. **提供清晰的產品描述**：越詳細越好
2. **從專案目錄運行**：讓 Claude 分析程式碼以更好理解

## 總結

### 成功要素

1. **脈絡理解**：深入分析您的產品和市場
2. **智慧搜尋**：尋找符合 ICP 的公司
3. **優先級排序**：明確的推薦而非隨機清單
4. **個人化**：每個潛在客戶的具體策略
5. **可行動性**：提供具體的下一步而非泛泛之談

### 時間節省

**手動方式**：
- 公司研究：20 分鐘/每家 × 20 = 6.5 小時
- 評估適配度：5 分鐘/每家 × 20 = 1.5 小時
- 起草接觸策略：15 分鐘/每家 × 20 = 5 小時
- **總計：13 小時**

**使用此 Skill**：
- 生成潛在客戶清單：10-15 分鐘
- 審查和優先排序：15-20 分鐘
- **總計：25-35 分鐘**

**節省：~95% 時間**

### 適用場景

- B2B 銷售潛在客戶開發
- 業務開發研究
- 合作夥伴識別
- 市場擴張規劃
- 競爭對手客戶分析
