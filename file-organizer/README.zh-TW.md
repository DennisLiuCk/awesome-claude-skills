# File Organizer - 正體中文說明文件

## Skill 概述

### 功能
File Organizer skill 是您的個人組織助手，透過理解檔案脈絡、尋找重複檔案、建議更好的結構以及自動化清理任務，智慧地組織您電腦中的檔案和資料夾。減少持續手動整理的認知負擔，保持數位工作空間的整潔。

### 解決的問題
使用者在管理電腦檔案時常面臨以下挑戰：

1. **下載資料夾混亂**：數百個未分類的檔案堆積
2. **找不到檔案**：檔案散落各處難以尋找
3. **重複檔案浪費空間**：同一檔案的多個副本佔用儲存空間
4. **資料夾結構不合理**：原有的組織方式已不適用
5. **缺乏組織習慣**：不知如何建立良好的檔案管理系統
6. **專案清理困難**：舊專案檔案混雜難以歸檔

這個 skill 自動化檔案組織過程，根據檔案類型、日期和內容做出智慧決策，無需手動整理的心力負擔。

## 核心功能分析

### 1. 分析當前結構

**檢視內容**：
- 總檔案和資料夾數量
- 檔案類型分布
- 大小分布和佔用空間
- 日期範圍
- 明顯的組織問題

**使用命令**：
```bash
# 列出目錄概況
ls -la [target_directory]

# 檢查檔案類型和大小
find [target_directory] -type f -exec file {} \;

# 識別最大檔案
du -sh [target_directory]/* | sort -rh | head -20

# 統計檔案類型
find [target_directory] -type f | sed 's/.*\.//' | sort | uniq -c | sort -rn
```

### 2. 尋找重複檔案

**識別方式**：
- 透過雜湊值尋找完全相同的檔案
- 找出同名檔案
- 識別相似大小的檔案

**重複檔案處理**：
- 顯示所有檔案路徑
- 顯示大小和修改日期
- 建議保留哪個（通常是最新或最佳命名的）
- **重要**：刪除前一定要求確認

### 3. 建議組織策略

**按類型分組**：
- 文件（PDF、DOCX、TXT）
- 圖片（JPG、PNG、SVG）
- 影片（MP4、MOV）
- 壓縮檔（ZIP、TAR、DMG）
- 程式碼/專案（包含程式碼的目錄）
- 試算表（XLSX、CSV）
- 簡報（PPTX、KEY）

**按目的分組**：
- 工作 vs. 個人
- 進行中 vs. 已歸檔
- 專案特定
- 參考資料
- 暫存/草稿檔案

**按日期分組**：
- 當前年份/月份
- 過往年份
- 非常舊（歸檔候選）

### 4. 自動化清理

**移動檔案**：
- 根據類型移至適當資料夾
- 保留原始修改日期
- 記錄所有移動以便復原

**重新命名**：
- 統一命名格式
- 移除下載檔名的雜亂部分
- 使用描述性名稱

**創建結構**：
- 建立邏輯資料夾層級
- 按需要分組
- 保持簡潔易導航

### 5. 維護脈絡理解

**智慧決策基於**：
- 檔案類型和副檔名
- 修改和創建日期
- 檔案名稱中的線索
- 檔案大小和內容
- 使用者的工作流程

### 6. 減少混亂

**識別可移除項目**：
- 6 個月以上未碰觸的舊檔案
- 明顯的暫存檔案
- 重複檔案
- 空資料夾
- 快取和暫存資料

### 典型使用場景

#### 場景 1：下載資料夾整理（來自 Justin Dielmann）

**使用者**：「我的下載資料夾有 500 多個檔案，一團混亂。幫我整理。」

**處理流程**：
1. **分析**：掃描 500 個檔案
2. **發現模式**：工作文件、個人照片、安裝程式、PDF 等
3. **提出結構**：
   ```
   Downloads/
   ├── Work/
   ├── Personal/
   ├── Installers/ (DMG、PKG 檔案)
   ├── Archive/
   └── ToSort/ (需要決定的項目)
   ```
4. **請求確認**
5. **執行**：根據內容和名稱智慧移動檔案
6. **結果**：500 個檔案 → 5 個有組織的資料夾

#### 場景 2：尋找並移除重複檔案

**使用者**：「在我的文件中尋找重複檔案，幫我決定保留哪些。」

**輸出**：
```markdown
# 找到 23 組重複檔案（總計 156 MB）

## 重複組 1：「proposal.pdf」
- `/Documents/proposal.pdf` (2.3 MB，修改：2024-03-15)
- `/Documents/old/proposal.pdf` (2.3 MB，修改：2024-03-15)
- `/Desktop/proposal.pdf` (2.3 MB，修改：2024-03-10)

**建議**：保留 `/Documents/proposal.pdf`（最新且位置正確）
刪除另外 2 個副本嗎？

[為所有重複項目繼續...]
```

#### 場景 3：重組專案資料夾

**使用者**：「檢視我的 ~/Projects 目錄並建議改進。」

**輸出**：
```markdown
# ~/Projects 分析

## 當前結構問題

- 無一致的命名慣例
- 部分專案在根目錄，其他在隨機子資料夾
- 重複資料夾（project-name、project-name-old、project-name-v2）

## 建議結構

```

├── Active/
│   ├── client-work/

3. 重新命名專案為一致格式：「client-name-project-name」

## 最佳實踐

### 使用技巧

1. **從小範圍開始**：先整理一個混亂的資料夾（如下載）建立信任
2. **定期維護**：每週對下載執行清理
3. **一致的命名**：重要檔案使用「YYYY-MM-DD - 描述」格式
4. **積極歸檔**：移動舊專案到歸檔而不是刪除
5. **分離進行中和已完成**：在進行中和已歸檔工作之間保持清晰界限
6. **信任流程**：讓 Claude 處理檔案歸屬的認知負擔

### 資料夾命名建議

**清晰描述性**：
- 使用清楚、描述性的名稱
- 避免空格（使用連字號或底線）
- 具體化：「client-proposals」而非「docs」
- 使用前綴排序：「01-current」、「02-archive」

### 檔案命名建議

**一致格式**：
- 包含日期：「2024-10-17-meeting-notes.md」
- 具描述性：「q3-financial-report.xlsx」
- 避免版本號（改用版本控制）
- 移除下載殘留：「document-final-v2 (1).pdf」→「document.pdf」

### 何時歸檔

**歸檔時機**：
- 6 個月以上未碰觸的專案
- 可能稍後需要參考的已完成工作
- 遷移到新系統後的舊版本
- 不想刪除但猶豫的檔案（先歸檔）

### 維護建議

**週期性任務**：
- **每週**：整理新下載檔案
- **每月**：檢視並歸檔已完成專案
- **每季**：檢查新的重複檔案
- **每年**：歸檔舊檔案

## 總結

### 成功要素

1. **智慧分析**：理解檔案脈絡和用途
2. **自動化處理**：減少手動整理的認知負擔
3. **安全第一**：重要操作前請求確認
4. **可維護性**：提供持續組織的建議
5. **靈活適應**：支援多種組織策略

### 時間節省

**手動方式**：
- 分析檔案：1-2 小時
- 決定組織方式：30 分鐘
- 移動和重新命名：2-3 小時
- **總計：4-6 小時**

**使用此 Skill**：
- 分析和建議：5 分鐘
- 審查計畫：5 分鐘
- 自動執行：10 分鐘
- **總計：20 分鐘**

### 適用場景

- 下載資料夾清理
- 專案目錄重組
- 重複檔案移除
- 桌面整理
- 照片組織
- 工作/個人分離
- 歸檔舊專案

### 認知負擔減輕

這個 skill 的最大價值不僅是節省時間，更是**減少決策疲勞**。不再需要：
- 記住檔案應該放在哪裡
- 每次都重新決定組織策略
- 擔心檔案找不到
- 手動追蹤重複檔案

讓 Claude 處理組織邏輯，您專注於重要的工作。
