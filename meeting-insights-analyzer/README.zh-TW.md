# Meeting Insights Analyzer - 正體中文說明文件

## Skill 概述

### 功能
Meeting Insights Analyzer skill 分析會議逐字稿和錄音，揭露行為模式、溝通洞察和可行動的回饋。識別當您迴避衝突、使用填充詞、主導對話或錯過傾聽機會時。非常適合尋求改善溝通和領導技能的專業人士。

### 解決的問題
專業人士在自我了解和溝通改進時常面臨以下挑戰：

1. **盲點**：不知道自己的溝通習慣
2. **缺乏具體回饋**：籠統的建議難以行動
3. **主觀自我評估**：很難客觀評價自己
4. **模式難識別**：單一會議看不出長期趨勢
5. **改進追蹤困難**：不知道是否有進步
6. **無量化數據**：缺少具體指標來衡量表現

這個 skill 將會議逐字稿轉化為可行動的洞察，幫助您成為更有效的溝通者和領導者。

## 核心功能分析

### 1. 模式識別

**行為模式**：
- **衝突迴避**：間接溝通、迴避困難對話
- **說話比例**：您 vs. 他人的說話時間
- **提問模式**：問題 vs. 陳述比例
- **主動傾聽**：建立在他人想法上
- **決策方式**：指令型 vs. 協作型

**溝通習慣**：
- 填充詞頻率（「嗯」、「呃」、「像是」）
- 打斷他人或被打斷的次數
- 措辭模式（避險語言、自信表達）
- 反應速度和深思熟慮

### 2. 溝通分析

**效果評估**：
- **清晰度和直接性**：訊息是否明確
- **填充詞使用**：「嗯」、「呃」、「你知道」等
- **語氣和情感**：情緒模式
- **會議控制**：引導和促進能力

**具體範例**：
每個模式都附有：
- 實際引用（附時間戳記）
- 為何重要的說明
- 如何改進的建議

### 3. 衝突迴避分析

**識別方式**：
- 避險語言（「也許」、「有點」、「我認為」）
- 間接措辭而非直接請求
- 緊張時轉換話題
- 不置可否的同意（「是的，但是...」）
- 不處理明顯問題

**輸出範例**：
```markdown
### 衝突迴避

**發現**：在 7 次會議中使用間接溝通 8 次

**範例 1：與 Sarah 的 1:1** - 00:14:32

**發生什麼**：
> 「所以，我在想...也許我們可以，像是，可能考慮
> 再看一下時間表？我的意思是，如果你認為這有道理。
> 但無論你認為最好！」

**為何重要**：
您需要處理 Sarah 的專案落後進度，但避險語言
（「也許」、「可能」、「如果你認為」）和最後的推
託（「無論你認為」）讓她很容易錯過緊急性。

**更好的方法**：
「Sarah，專案落後兩週進度。我們需要討論什麼阻礙
了進度，今天就制定新的時間表。」
```

### 4. 說話統計

**量化指標**：
- 平均說話時間（會議的 %）
- 提出的問題數量（每次會議平均）
- 填充詞（每分鐘）
- 打斷次數（給予/接受，每次會議）

### 5. 趨勢追蹤

**時間比較**：
- Q1 vs. Q2 模式變化
- 改進領域進展
- 新出現的挑戰
- 技能發展軌跡

### 6. 可行動回饋

**每個洞察包含**：
1. **觀察到什麼**：具體行為
2. **為何重要**：影響或錯失機會
3. **如何改進**：具體可行動建議

## Claude Code 使用方式

### 觸發時機

Claude Code 會在以下情況使用此 Skill：

1. **分析溝通模式**
   ```
   分析此資料夾中的所有會議，告訴我何時迴避衝突
   ```

2. **審查過去會議**
   ```
   查看我過去一個月的會議，識別我的溝通模式
   ```

3. **比較時期**
   ```
   比較這兩個會議資料夾之間我的引導風格
   ```

4. **多維度分析**
   ```
   分析此資料夾中所有逐字稿：
   1. 識別我何時打斷他人
   2. 計算我的說話比例
   3. 找出我迴避直接回饋的時刻
   4. 追蹤我的填充詞使用
   5. 顯示良好主動傾聽的範例
   ```

### 典型使用場景

#### 場景 1：衝突迴避分析（受 Dan Shipper 啟發）

**使用者**：「我下載所有會議錄音並放在資料夾中。告訴我所有我微妙迴避衝突的時刻。」

**輸出**：
```markdown
# 衝突迴避模式

在 15 次會議中找到 23 個您使用間接溝通或迴避直接
處理緊張的案例。

## 模式：對關鍵回饋的避險

**頻率**：7 次會議中出現 8 次

**範例 1：與 Sarah 的 1:1** - 00:14:32

[詳細分析...]

## 建議

1. **在第一句話中直接點名問題**
2. **移除避險詞**如「也許」、「有點」、「某種」
3. **問具體問題**而非暗示
4. **安排困難對話**而非隨意提起
```

#### 場景 2：領導引導分析

**使用者**：「分析我的團隊會議，告訴我關於我的引導風格。」

**輸出**：提供以下洞察：
- 您說話 vs. 團隊成員（60% vs. 40%）
- 您問問題還是做陳述（3:1 比例）
- 如何處理分歧（傾向過快解決）
- 誰說話最少以及您是否引導他們
- 良好和錯失引導時刻的範例

#### 場景 3：個人發展追蹤

**使用者**：「比較我 Q1 vs. Q2 的會議，看我的傾聽技能是否有改善。」

**輸出**：創建比較分析顯示：
- 打斷減少（每次會議 8 次 → 3 次）
- 澄清問題增加（每次會議 2 次 → 7 次）
- 建立在他人想法上的改善
- 顯示差異的具體範例
- 仍有成長空間的領域

## 設定技巧

### 獲取會議逐字稿

**從 Granola**（Lenny's newsletter 訂閱者免費）：
- Granola 自動轉錄您的會議
- 將逐字稿匯出到資料夾
- 將 Claude Code 指向該資料夾

**從 Zoom**：
- 啟用雲端錄製與轉錄
- 會議後下載 VTT 或 SRT 檔案
- 存放在專用資料夾

**從 Google Meet**：
- 使用 Google Docs 自動轉錄
- 將逐字稿文件保存到資料夾
- 下載為 .txt 檔案或給 Claude Code 存取權

**從 Fireflies.ai、Otter.ai 等**：
- 批量匯出逐字稿
- 存放在本地資料夾
- 對資料夾運行分析

### 最佳實踐

1. **一致命名**：使用 `YYYY-MM-DD - 會議名稱.txt` 格式
2. **定期分析**：每月或每季審查趨勢
3. **具體查詢**：一次專注於一個行為以獲得深度
4. **隱私**：將敏感會議數據保留在本地
5. **行動導向**：一次專注於一個改進領域

## 總結

### 成功要素

1. **客觀數據**：基於實際逐字稿而非記憶
2. **具體範例**：附時間戳記的真實引用
3. **模式識別**：長期趨勢而非個別事件
4. **可行動回饋**：具體建議而非泛泛之談
5. **進度追蹤**：時間比較顯示改善

### 適用場景

- 改善溝通技能
- 領導力發展
- 績效評估準備
- 教練團隊成員
- 自我意識提升
- 職業發展
