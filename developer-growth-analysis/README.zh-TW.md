# Developer Growth Analysis - 正體中文說明文件

## Skill 概述

### 功能
Developer Growth Analysis 分析您最近的 Claude Code 聊天歷史記錄，以識別編碼模式、開發差距和改進領域，從 HackerNews 策劃相關學習資源，並自動將個人化成長報告發送到您的 Slack 直接訊息。

### 解決的問題
開發者在個人成長方面面臨的挑戰：

1. **缺乏客觀反饋**：難以客觀評估自己的編碼習慣和技能差距
2. **盲點**：不知道自己重複犯同樣的錯誤或低效的做法
3. **學習方向不明**：不確定應該專注於哪些技能提升最有價值
4. **資源泛濫**：網路上有太多學習資源，難以找到真正相關的內容
5. **進度難以追蹤**：沒有系統化的方式追蹤技能成長
6. **等待 Code Review**：需要等待團隊 code review 才能獲得反饋

這個 skill 通過分析您的實際工作模式，提供數據驅動的個人化成長洞察和行動計劃。

## 核心功能

### 1. 讀取聊天歷史
- 訪問本地 Claude Code 聊天歷史（過去 24-48 小時）
- 從 `~/.claude/history.jsonl` 讀取數據
- 基於時間戳過濾相關條目
- 提取專案、訊息和貼上的內容

### 2. 識別開發模式
分析以下方面：
- **專案和領域**: 您在什麼類型的專案上工作？（後端、前端、DevOps、數據等）
- **使用的技術**: 對話中出現哪些語言、框架和工具？
- **問題類型**: 正在解決哪些類別的問題？（效能優化、除錯、功能實作、重構、設定/配置）
- **遇到的挑戰**: 您在什麼問題上掙扎？尋找重複問題、多次嘗試解決的問題、表明知識差距的問題
- **方法模式**: 您如何解決問題？（有條理、探索性、實驗性）

### 3. 檢測改進領域
識別 3-5 個具體領域，這些應該是：
- **具體的**（不是「提高編碼技能」這樣的模糊建議）
- **基於證據的**（基於實際聊天歷史）
- **可操作的**（可以採取的實際改進措施）
- **優先排序的**（最有影響力的優先）

範例：
- 「高級 TypeScript 模式（泛型、實用類型、類型守衛）- 您在 [特定專案] 中與類型安全掙扎」
- 「錯誤處理和驗證 - 我注意到您修補了幾個與缺少空值檢查相關的錯誤」
- 「非同步/等待模式 - 您最近的工作顯示一些競爭條件和時間問題」

### 4. 生成個人化報告
建立包含以下內容的綜合報告：
- 工作摘要
- 改進領域（優先排序）
- 每個領域的具體建議
- 觀察到的優勢
- 行動項目

### 5. 尋找學習資源
使用 Rube MCP 從 HackerNews 搜尋文章：
- 針對每個改進領域構建搜尋查詢
- 優先考慮高參與度的貼文（評論、點讚）
- 每個領域包含 2-3 篇最相關的文章
- 提供標題、日期、相關性描述和連結

### 6. 發送到 Slack DMs
自動將完整報告傳送到您的 Slack：
- 使用 Rube MCP 傳送訊息
- 將報告分為邏輯部分
- 格式化為結構良好的 Slack 訊息
- 包含可點擊的學習資源連結

## 設計策略

### 1. 數據驅動
基於實際工作模式而非假設：
- 分析真實的聊天歷史
- 使用時間戳進行準確過濾
- 尋找可量化的模式
- 提供具體證據

### 2. 可操作的洞察
提供實用而非理論的建議：
- 具體的技能領域
- 實際的學習資源
- 估計的學習時間
- 優先排序的行動項目

### 3. 個人化
針對您的實際工作：
- 基於您使用的技術
- 考慮您的專案類型
- 匹配您的挑戰
- 保持相關性

### 4. 持續追蹤
設計用於定期使用：
- 建議每週運行一次
- 追蹤隨時間的改進軌跡
- 專注於一個領域直到進步
- 重新評估後查看變化

## Claude Code 使用方式

### 觸發時機
Claude Code 會在以下情況使用此 Skill：

1. **基本分析請求**
   ```
   分析我最近聊天中的開發者成長
   ```

2. **特定時間範圍**
   ```
   分析我今天的工作並建議改進領域
   ```

3. **週期性審查**
   ```
   給我本週的開發者成長報告
   ```

### 典型工作流程

1. **讀取歷史**: Claude 訪問 `~/.claude/history.jsonl`
2. **過濾數據**: 根據時間戳（過去 24-48 小時）
3. **分析模式**: 提取專案、技術、問題類型、挑戰
4. **識別差距**: 找出 3-5 個改進領域
5. **生成報告**: 建立結構化的成長報告
6. **搜尋資源**: 從 HackerNews 找相關文章
7. **發送到 Slack**: 自動傳遞到您的 DM

## 使用的 Prompt 策略

### 1. 詳細的 6 步流程
提供明確的指令：
1. 訪問聊天歷史
2. 分析工作模式
3. 識別改進領域
4. 生成報告
5. 搜尋學習資源
6. 發送到 Slack DMs

### 2. 數據結構說明
清楚描述 history.jsonl 格式：
- `display`: 使用者訊息/請求
- `project`: 正在工作的專案
- `timestamp`: Unix 時間戳（毫秒）
- `pastedContents`: 任何程式碼或貼上的內容

### 3. 分析框架
提供多維度分析視角：
- 專案和領域
- 使用的技術
- 問題類型
- 遇到的挑戰
- 方法模式

### 4. 品質標準
明確改進領域應該具備的特徵：
- 具體性
- 證據基礎
- 可操作性
- 優先排序

## 使用的工具（Tools）

此 Skill 主要依賴以下 Claude Code 工具：

### 1. **Read Tool**
- 讀取 `~/.claude/history.jsonl`
- 分析聊天歷史數據
- 提取模式和證據

### 2. **Bash Tool**
- 獲取當前時間戳
- 過濾日期範圍
- 處理 JSONL 檔案

### 3. **MCP Tools (Rube)**
- **RUBE_SEARCH_TOOLS**: 從 HackerNews 搜尋文章
- **RUBE_MANAGE_CONNECTIONS**: 初始化 Slack 認證
- **RUBE_MULTI_EXECUTE_TOOL**: 發送格式化的 Slack 訊息

### 4. **Write Tool**
- 生成報告檔案
- 建立結構化輸出
- 格式化 Markdown

## 專業技巧

### 使用建議
1. **每週運行一次**: 追蹤隨時間的改進軌跡
2. **一次專注一個領域**: 在移到下一個之前專注幾天
3. **使用學習資源**: 將它們作為學習指南工作
4. **重新評估進度**: 專注一週後重新訪問報告
5. **策劃的相關性**: 資源是為您的實際工作策劃的，不是泛泛的主題

### 最佳實踐
- 將報告視為成長指南，不是批評
- 慶祝觀察到的優勢
- 對行動項目設定實際時間表
- 在 Slack 中保存報告以便隨時參考
- 與導師分享以獲得外部反饋（如果需要）
