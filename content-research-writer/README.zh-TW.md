# Content Research Writer - 正體中文說明文件

## Skill 概述

### 功能
Content Research Writer 作為您的寫作夥伴，協助您研究、大綱、草稿和完善內容，同時保持您獨特的聲音和風格。它將寫作過程從獨自努力轉變為協作夥伴關係。

### 解決的問題
內容創作者在寫作過程中面臨的挑戰：

1. **研究耗時**：查找可靠資訊和引用來源需要大量時間
2. **結構規劃困難**：將想法組織成連貫的大綱不容易
3. **開頭困難**：寫出吸引人的開場白和引言具有挑戰性
4. **缺乏即時反饋**：在寫作過程中難以獲得建設性反饋
5. **引用管理複雜**：正確格式化和管理參考文獻很繁瑣
6. **孤獨的寫作過程**：獨自寫作缺乏互動和啟發

這個 skill 提供全程協作式寫作支援，從大綱到最終潤飾，讓您專注於創作而非瑣事。

## 核心功能

### 1. 協作式大綱
- 幫助將想法結構化為連貫的大綱
- 確保邏輯流程
- 識別研究空白
- 標記需要深入探討的部分

### 2. 研究協助
- 尋找相關資訊
- 發現可信來源
- 提取關鍵事實、引用和數據
- 以請求的格式添加引用

### 3. 開場白優化
- 分析當前開場白的效果
- 提供多個改進選項
- 說明每個選項為何有效
- 確保創造好奇心和價值承諾

### 4. 逐段反饋
- 評估每個段落的寫作
- 提供清晰度建議
- 改善流暢性
- 強化論證和證據

### 5. 聲音保持
- 維持您的寫作風格和語氣
- 提供建議而非替換
- 尊重您的選擇
- 增強而非覆蓋您的聲音

### 6. 引用管理
- 處理多種引用格式（內聯、編號、腳註）
- 維護運行中的引用清單
- 確保引用完整性和一致性

### 7. 迭代精煉
- 通過多個草稿幫助改進
- 提供全面的最終審查
- 發布前檢查清單

## 設計策略

### 1. 以使用者為中心
專注於保持作者的聲音：
- 學習他們的風格
- 提供選項而非指令
- 匹配語氣（正式、休閒、技術、友好）
- 尊重作者偏好

### 2. 結構化流程
提供清晰的 8 步寫作工作流程：
1. 了解寫作專案
2. 協作式大綱
3. 進行研究
4. 改進開場白
5. 提供逐段反饋
6. 保持作者聲音
7. 引用管理
8. 最終審查和潤飾

### 3. 多格式支援
適用於各種寫作類型：
- 部落格文章
- 電子報
- 技術教程
- 思想領導文章
- 案例研究

## Claude Code 使用方式

### 觸發時機
Claude Code 會在以下情況使用此 Skill：

1. **開始新寫作專案**
   ```
   幫我為關於 [主題] 的文章建立大綱
   ```

2. **需要研究協助**
   ```
   研究 [特定主題] 並在我的大綱中添加引用
   ```

3. **改進開場白**
   ```
   這是我的開場。幫我讓開場白更吸引人。
   ```

4. **獲取段落反饋**
   ```
   我剛完成「為什麼這很重要」部分。審查並給予反饋。
   ```

5. **最終審查**
   ```
   審查完整草稿的流暢性、清晰度和一致性。
   ```

### 典型工作流程

#### 部落格文章工作流程
1. 一起制定大綱
2. 研究關鍵點
3. 撰寫開場 → 獲得反饋
4. 撰寫主體段落 → 每段反饋
5. 撰寫結論 → 最終審查
6. 潤飾和編輯

#### 電子報工作流程
1. 討論開場白想法
2. 快速大綱（較短格式）
3. 一次完成草稿
4. 審查清晰度和連結
5. 快速潤飾

#### 技術教程工作流程
1. 大綱步驟
2. 撰寫程式碼範例
3. 添加說明
4. 測試指示
5. 添加疑難排解部分
6. 最終準確性審查

```markdown

## 簡介
- 背景和上下文
- 問題陳述
- 本文涵蓋內容

## 主要部分

### 第 1 部分：[標題]
- 關鍵點 A
- 關鍵點 B
- 範例/證據
- [需要研究：特定主題]

### 第 2 部分：[標題]
- 關鍵點 C
- 關鍵點 D
- 需要數據/引用

## 結論
- 主要點摘要
- 行動呼籲
- 最後思考

## 研究待辦事項
- [ ] 查找關於 [主題] 的數據
```

```markdown

- [優點 3]

### 清晰度
- [具體問題] → [建議修正]

### 流暢性
- [過渡問題] → [更好的連接]
- [段落順序] → [建議重新排序]

### 證據
- [需要支持的聲明] → [添加引用或範例]
- [泛泛陳述] → [更具體化]

## 具體行編輯

原文:
> [草稿的確切引用]

建議:
> [改進版本]

為什麼: [說明]

## 思考問題
```

## 專業技巧

### 為了研究
- 在引用前驗證來源
- 盡可能使用最新數據
- 平衡不同觀點
- 連結到原始來源

### 為了反饋
- 具體說明您想要什麼：「這太技術性了嗎？」
- 分享您的顧慮：「我擔心這部分拖沓」
- 提出問題：「這邏輯流暢嗎？」
- 請求替代方案：「還有什麼方法可以解釋這個？」

### 為了聲音
- 分享您寫作的範例
- 指定語氣偏好
- 指出好的匹配：「這聽起來像我！」
- 標記不匹配：「對我的風格來說太正式了」
